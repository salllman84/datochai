Deep Learning via LSTM Neural Networks  
To resolve the long -term sequential dependencies inherent in massive lottery draw 
histories, the computational architecture deploys Long Short -Term Memory (LSTM) 
networks. LSTMs are clinically designed to circumvent the vanishing gradient 
problem found in standard Recurrent Neural Networks (RNNs) when analyzing data 
spanning tens of thousands of temporal steps. The unique engineering of the LSTM 
cell relies on three regulatory gates. The Forget Gate determines which historical 
noise to discard via the mathe matical formulation: $$f_t = \sigma(W_{f,x} x_t + 
W_{f,h} h_{t -1} + b_f)$$Simultaneously, the Input Gate captures newly discovered 
anomalies from recent draws to create candidate memories:  
$$i_t = \sigma(W_{i,x}x_t + W_{i,h}h_{t -1} + b_i)$$  
$$\tilde{c}_t = \tanh(W_c[x_t, h_{t -1}] + b_c)$$  
Ultimately, the Cell State synchronizes these validated inputs, granting the network 
the perceptual intelligence to track deeply embedded cyclic volatility across the 
Malaysian 4D markets.  
Classical Time -Series and Additive Prophet Integration  
To prevent the neural network from overfitting to microscopic anomalies, the system 
integrates the Autoregressive Integrated Moving Average (ARIMA) model to isolate 
macro -linear trends and establish stationary plateaus. The model mathematically 
binds autor egressive relationships to historical events while utilizing moving average 
parameters to correct for residual statistical errors.  
In tandem, the architecture deploys the Meta Prophet framework, which processes 
data through a highly rigid additive formulation:  
$$y_t = g(t) + s(t) + h(t) + \epsilon_t$$

The definitive predictive array published to the DatoChai platform is synthesized 
through a Stacked Hybrid Ensemble Network. The ARIMA and Prophet models

NextAuth.js Integration and Collision Resolution  
The system integrates the Auth.js (formerly NextAuth.js) framework, paired directly 
with the Prisma Adapter, to manage session states and bridge authentication events 
securely to the PostgreSQL database. To systematically dismantle onboarding 
friction, del egated social login modalities (Google, Apple, Microsoft) are prominently

A critical architectural vulnerability in multi -provider ecosystems is identity collision. If 
a user authenticates via a Google JSON Web Token (JWT) utilizing an email 
address already registered within the internal database via standard credentials, 
genera ting a duplicate account fragments the user experience and breaks relational 
integrity. The architecture resolves this by extracting the verified email claim from the 
incoming JWT. Upon detecting a collision, the system suspends the OAuth 
sequence and dyna mically surfaces a security prompt, requiring the user to verify 
their original password. Upon successful verification, the backend updates the 
account domain tuple, seamlessly merging the OAuth authenticator ID into the 
existing unified user profile, ther eby neutralizing account takeover vulnerabilities.

Optimization  
To realize the stated performance objectives and ensure sub -second global

PM2 Process Clustering and Daemon Decoupling  
The bespoke Next.js Node application is deployed in a multi -threaded cluster mode 
via the PM2 process manager, ensuring it completely saturates all available CPU 
cores on the Hostinger Virtual Private Server. A well -documented architectural risk in 
monolit hic Node.js environments is memory leakage when executing heavy 
background scheduling tasks. To definitively mitigate this, the node -cron automation 
daemon is deliberately decoupled into an entirely isolated PM2 fork ( cron-worker.js ). 
This physical separation ensures that heavy background computational tasks —such 
as archiving massive historical datasets or routing outbound webhooks —cannot 
degrade the performance or memory availability of the primary web delivery threads.

Legacy polymorphic metadata structures have been discarded. The database
